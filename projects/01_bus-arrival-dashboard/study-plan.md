# 博多→九大 通学時間ダッシュボード — 学習計画

## 自己診断: ピンと来ていない技術

- **Chart.js**: 設定をコピペで動かしたが、各オプション（`scales`, `plugins`, `datasets` の構造など）が何を意味しているか説明できない
- **pandas（集計スクリプト）**: Claude が書いた `groupby` / `pivot` あたりの処理が読めていない。CSV → JSON への変換ロジックを自分の手で再現できない自信がある

## 実運用で突っ込まれそうな点

1. **データの鮮度**: 2021年の道路交通センサスのまま。最新化の仕組み（再集計バッチ）がない
2. **カバレッジ**: 国道202号のみ。「他路線は？」「他キャンパスは？」と必ず聞かれる
3. **再現性**: 集計スクリプトが手元の Python 実行環境前提。誰でも再現できる手順書 / CI がない

→ 関連用語: **CI/CD**, **再現可能性（reproducibility）**, **データパイプライン**

## デリバリー前提のチェック棚卸し

| 観点 | 学生の回答 | 関連する専門用語 |
|---|---|---|
| 認証・認可 | 全員に同じ画面なので何もしていない。将来「お気に入りルート保存」を足すなら必要 | authentication / authorization |
| 入力の扱い | タイムピッカーとセレクトボックスのみ。自由入力欄は今のところなし | XSS / 入力バリデーション |
| データの出どころ | 道路交通センサスは政府データ。Chart.js は MIT。pandas のライセンスは未確認 | OSS ライセンス（MIT / BSD / Apache / GPL） / 政府標準利用規約 |
| 個人情報・法令 | 現状なし。「現在地から最寄り出発地を自動選択」を足すと位置情報が個人情報になりうる | 個人情報保護法 / プライバシーポリシー / 取得目的の通知 |
| コスト・スケール | 静的サイトなので無料枠で足りそう。バズった場合の帯域課金が未把握 | 従量課金 / Quota / レートリミット / CDN |
| 障害・運用 | 落ちても誰かに指摘されるまで気付かない。バックアップという概念は未検討 | モニタリング / アラート / オブザーバビリティ |

## 学習ロードマップ

### 領域1: OSS ライセンス

- **なぜ学ぶ**: pandas など仕事で使うライブラリの扱いが分かっていないと、社内利用や商用化のときに法務でつまずきそう
- **入門**: 「OSS ライセンスのキホン」 — https://www.osslab.tokyo/oss-license-basics
- **公式（一次情報）**: SPDX License List — https://spdx.org/licenses/

### 領域2: 個人情報保護法

- **なぜ学ぶ**: 位置情報を扱う機能を足したいので、最低限の取り扱いルールを押さえたい
- **入門**: 個人情報保護委員会「はじめての個人情報保護法」 — https://www.ppc.go.jp/personalinfo/beginner/
- **公式（一次情報）**: 個人情報保護法ガイドライン（通則編） — https://www.ppc.go.jp/personalinfo/legal/guidelines_tsusoku/

### 領域3: モニタリング / アラート

- **なぜ学ぶ**: 運用してみたい気持ちはあるが、「落ちたら気付ける状態」を作る経験がない
- **入門**: 「無料ではじめる Web サイト死活監視（UptimeRobot 編）」 — https://uptimerobot.com/blog/website-monitoring-getting-started/
- **公式（一次情報）**: Google SRE Book - Monitoring Distributed Systems — https://sre.google/sre-book/monitoring-distributed-systems/
