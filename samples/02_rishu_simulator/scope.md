# スコープ設計書: 履修登録シミュレータ (jiji)

## 1. 概要

| 項目 | 内容 |
|---|---|
| 課題 | 履修登録のときに複数の候補科目から選ぶのだが、時間割の衝突や単位数の合計を手作業で確認するのが大変で、プランを比較するのも面倒 |
| 解決策 | 履修候補をCSVに書き出すと、時間割の衝突・週マップ・単位数サマリをターミナルに表示するCLIツール (`jiji`) を作る |
| 対象ユーザー | 履修登録を控えた大学生（学部問わず） |
| アーキテクチャ | 単一のPythonパッケージ。CSVを読み込んでターミナルに整形出力するだけのシンプル構成 |

## 2. 入力フォーマット

履修候補は1つのCSVファイルにまとめる。各行が1科目。

### courses.csv

```csv
id,name,day,period,credits,category,priority
CS101,プログラミング基礎,月,1,2,必修,high
CS102,データ構造,火,2,2,必修,high
CS201,アルゴリズム,水,3,2,選択必修A,medium
CS202,データベース,水,3,2,選択必修A,medium
CS203,ネットワーク,木,1,2,選択必修A,low
BIZ101,経営学入門,月,2,2,選択,medium
BIZ102,マーケティング,火,1,2,選択,high
LANG101,英語,金,2,1,必修,high
```

### カラム定義

| カラム | 必須 | 内容 |
|---|---|---|
| `id` | ✓ | 科目コード（ユニーク） |
| `name` | ✓ | 科目名 |
| `day` | ✓ | 曜日（月/火/水/木/金） |
| `period` | ✓ | 時限（1〜5の整数） |
| `credits` | ✓ | 単位数 |
| `category` | ✓ | 区分（必修/選択必修A/選択 など、自由文字列） |
| `priority` | — | 取りたさ（high/medium/low）。未指定は medium |

### 注意点

- UTF-8 エンコーディング想定
- `category` の値は自由。ツール側は集計のキーとして扱うだけで、意味は解釈しない
- `priority` は衝突時の推奨判断にのみ使う（ストレッチ機能）

## 3. 機能一覧

### MVP（必ず作る）

| # | 機能 | 説明 |
|---|---|---|
| 1 | `jiji check` | CSVを読み込み、衝突検出 → 週マップ → 単位数サマリを順に表示 |
| 2 | 衝突検出 | 同じ `(day, period)` に複数科目がある場合に警告 |
| 3 | 週マップ表示 | 曜日×時限の表を Rich の Table で描画（衝突セルはハイライト） |
| 4 | 単位数サマリ | category 別の単位数合計を表示 |

### ストレッチ（時間があれば）

| # | 機能 | 説明 |
|---|---|---|
| 5 | `jiji diff planA.csv planB.csv` | 2プランを比較。差分科目 + 単位数比較を表示 |
| 6 | priority ベースの推奨 | 衝突時に priority を見て「こちらを残すと良い」をサジェスト |
| 7 | 時限マージ対応 | `period: "3-4"` で2コマ連続の科目を扱う |
| 8 | 集中講義対応 | `day: 集中` を特別扱い（週マップ外に別枠表示） |

## 5. 技術スタック

| レイヤー | 技術 | 選定理由 |
|---|---|---|
| 言語 | Python 3.10+ | code-server に標準で入っている。CSV処理が簡潔 |
| CLIフレームワーク | Click | デコレータでサブコマンド (`check` / `diff`) を定義できる |
| ターミナル装飾 | Rich | 表・色付き出力が簡単。結果の見栄えがよくなる |
| パッケージ管理 | uv | 高速。`uv pip install -e .` でコマンド登録 |

## 7. 実装ロジック

### ステップ1: CSVの読み込みと正規化

- `csv.DictReader` で読み込み
- `period` / `credits` を int に変換
- `priority` 未指定は `medium`
- `day` は `月〜金` のみ許容（それ以外はエラー）

### ステップ2: 衝突検出

- `(day, period)` をキーにして科目をグルーピング
- 同じキーに2件以上あれば衝突
- 衝突リストを `[(day, period, [course, course, ...]), ...]` の形で返す

### ステップ3: 週マップ描画

- 曜日（月〜金）× 時限（1〜5）の2次元テーブルを作る
- 各セルに `course.id` を入れる
- 衝突セルは `[衝突]` 表示 + 赤色でハイライト（Rich の style）

### ステップ4: 単位数サマリ

- `category` ごとに `credits` を合計
- 衝突がある場合は「衝突解消後」の見込みも出す
  - 衝突している科目のうち priority が最も高いもの1つだけ残した場合の単位数

### ステップ5: 最終整形

- 衝突 → 週マップ → サマリ の順に出力
- 衝突がなければ冒頭で `[OK] 時間割の衝突はありません` と表示

## 8. 実装計画（2コマ = 180分）

### 1コマ目（90分）: 読み込み + 衝突検出 + 週マップ

| 時間 | やること |
|---|---|
| 0〜20分 | プロジェクト初期化（pyproject.toml, uv セットアップ, サンプルCSV作成） |
| 20〜40分 | loader.py: CSV読み込み、型変換、バリデーション |
| 40〜60分 | conflict.py: 同時限の衝突検出ロジック |
| 60〜90分 | timetable.py: 週マップの Rich Table 描画、`jiji check` コマンドの雛形 |

### 2コマ目（90分）: サマリ + 仕上げ

| 時間 | やること |
|---|---|
| 0〜30分 | summary.py: カテゴリ別単位数集計、衝突解消後の見込み表示 |
| 30〜50分 | cli.py: 警告 → 週マップ → サマリ を一貫して表示、色分けの調整 |
| 50〜70分 | 全体の動作確認、エラー処理（CSV不正、必須カラム欠落など） |
| 70〜90分 | ストレッチ機能（`jiji diff` または priority ベース推奨のどちらか） |

## 9. スコープ外（やらないこと）

- 大学公式のシラバスからの自動取り込み（手書きCSV前提）
- 卒業要件の自動判定（「選択必修Aから4単位必要」等の要件定義は扱わない）
- GUI / Web UI（ターミナル出力のみ）
- シラバス情報の検索・科目レコメンド
- 前期/後期の同時管理（1学期分のみ）
- LLM による自然言語応答（構造化出力で完結させる）
- データの永続化（実行ごとにCSVを読み直すだけ）
