# jiji — 履修登録シミュレータ

履修候補を書き出したCSVを読み込み、**時間割の衝突検出 / 週マップ / 単位数サマリ**をターミナルに表示するCLIツールです。

## できること

- `(曜日, 時限)` が重複する科目の衝突を検出
- 曜日 × 時限の週マップを表形式で表示（衝突セルは赤ハイライト）
- `category`（必修 / 選択 等）別の単位数合計と、衝突解消後の見込みを算出

## 必要なもの

- Python 3.10 以上
- [uv](https://docs.astral.sh/uv/)（パッケージ管理）

## セットアップ

```bash
cd samples/02_rishu_simulator
uv sync
```

`uv sync` で `.venv` が自動作成され、`click` と `rich` がインストールされます。`jiji` コマンドも同時に登録されます。

## 使い方

### 基本: `jiji check`

```bash
uv run jiji check samples/courses.csv
```

## CSV フォーマット

1行1科目で、以下の列を必ず入れてください。

| 列名       | 必須 | 内容                                   |
| ---------- | ---- | -------------------------------------- |
| `id`       | ✓    | 科目コード（ユニーク）                 |
| `name`     | ✓    | 科目名                                 |
| `day`      | ✓    | 曜日（`月` / `火` / `水` / `木` / `金`）|
| `period`   | ✓    | 時限（1〜5の整数）                     |
| `credits`  | ✓    | 単位数（整数）                         |
| `category` | ✓    | 区分（`必修` / `選択必修A` / `選択` 等、自由文字列） |
| `priority` | —    | 取りたさ（`high` / `medium` / `low`）。未指定は `medium` |

- エンコーディングは **UTF-8**
- `category` は集計キーとして扱うだけで、意味は解釈しません
- `priority` は衝突解消後の見込み（最優先を1件残す）に使います

### 最小サンプル

```csv
id,name,day,period,credits,category,priority
CS101,プログラミング基礎,月,1,2,必修,high
CS102,データ構造,火,2,2,必修,high
CS201,アルゴリズム,水,3,2,選択必修A,medium
CS202,データベース,水,3,2,選択必修A,medium
```

動作確認用に `samples/courses.csv`（衝突あり）と `samples/planB.csv`（衝突なし）を同梱しています。

## ディレクトリ構成

```
02_rishu_simulator/
├── pyproject.toml          # click + rich, entry_point: jiji.cli:main
├── README.md               # このファイル
├── src/jiji/
│   ├── __init__.py
│   ├── cli.py              # Click エントリポイント
│   ├── loader.py           # CSV の読み込み + バリデーション
│   ├── conflict.py         # (day, period) の衝突検出
│   ├── timetable.py        # Rich Table で週マップを描画
│   └── summary.py          # category 別単位数集計
└── samples/
    ├── courses.csv         # 動作確認用（衝突あり）
    └── planB.csv           # 動作確認用（衝突なし）
```

## トラブルシュート

- `uv: command not found` → [uv のインストール手順](https://docs.astral.sh/uv/getting-started/installation/) を参照
- `読み込みエラー: 必須カラムが不足しています: ...` → 上のカラム定義を確認してください
- `読み込みエラー: day は 月/火/水/木/金 のいずれかです` → `day` 列の値を見直してください（全角ひらがな・漢字のみ許容）
